# 课程页模板

本文件总结站内课程页的常见结构，供新建课程页时直接参考。

## 🧱 稳定结构

站内绝大多数课程页都遵循同一主骨架：

1. 一级标题：`# 课程名`
2. `## 课程简介`
3. `## 课程资源`

在当前仓库里，这两个二级标题几乎是所有课程页的共同部分。额外小节可以按内容需要增补，但不建议一开始就发明很多新结构。

## ✏️ 写作要求

- 先用若干条目写清课程的基本信息，再写 1-3 段正文介绍。
- 信息项尽量复用站内已有标签，不要随意改名。
- 正文重点写课程讲什么、难点在哪里、适合什么背景的人学、和同类课程相比有什么特点。
- 资源区优先放课程网站、视频、教材、作业、社区笔记等直接可用的入口。
- 如果内容继承或改写自外部站点，请像站内已有页面一样在开头注明来源。

常用信息项如下，按需选用即可：

- `所属大学`
- `授课教师` / `任课教师`
- `先修要求`
- `编程语言`
- `课程难度`
- `预计学时`
- `课程代码`
- `学分/学时`
- `修读要求`

资源区常用条目如下，按需填写：

- `课程网站`
- `课程视频`
- `课程教材`
- `课程作业`
- `课程笔记（社区整理）`

常见可选小节如下；只有在确实有内容时再加：

- `## 讲师信息`
- `## 中文配套资源`
- `## 同领域其他名课`
- `## 备注`

## 📄 模板一：完整课程页

适用于公开课、外校课程，或已经有人能写出完整评价的课程页。

```md

*注：本课程页继承自 [CS自学指南](https://example.com/)。*

## 课程简介

- 所属大学：
- 授课教师：
- 先修要求：
- 编程语言：
- 课程难度：🌟🌟🌟
- 预计学时：

这里先用 1-3 段正文说明课程主线、覆盖内容、难点和适合人群。

如果这门课和站内另一门课有明显衔接关系，也可以在这里顺手说明。

## 课程资源

- 课程网站：
- 课程视频：
- 课程教材：
- 课程作业：

## 备注

可选。这里补充中文配套资料、避坑提醒、和其他课程的区别等。
```

## 🏫 模板二：复旦培养方案占位骨架

适用于培养方案里已经列出、但站内暂时还没有修过这门课的同学供稿的课程页。

```md
# 复旦：课程名

    复旦培养方案课程，本页为占位骨架。站内还没有修过这门课的同学供稿，欢迎通过[参与建设](相对路径按实际目录层级调整)补全课程简介、难度评价与学习资源。

## 课程简介

- 学分/学时：
- 修读要求：
- 任课教师：
- 课程难度：

## 课程资源

待补充。
```

## ✅ 提交前自查

- 文件名是否符合 `README.md` / `CONTRIBUTING.md` 里的命名规则。
- 父级 `index.md` 是否已经补上课程条目。
- `mkdocs.yml` 是否已经注册新文件路径。
- 页面里是否至少有 `## 课程简介` 和 `## 课程资源` 两个主小节。
- 信息项名称是否复用了站内已有写法。
