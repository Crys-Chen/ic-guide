# 作业规范：把方向页的「企业 / 科研院所 / 顶会顶刊」改成表格

你负责**一个**方向页。任务：把该页 `### 企业`、`### 科研院所`、`## 顶会顶刊` 三处的内容**改写成表格**。这是**纯排版转换**——不增删任何公司/机构/会议，不改任何链接、不改任何股价 badge。改完只动这三处，页面其余部分（含 `## 代表性机构` 标题、其下的引言 blockquote、`## 知识路径` 等）一律不动。

## 1. `### 企业` → 单表（地区作列）

把 `**国内**` / `**国外**` 两个 bullet 列表合并成一张表：

```
### 企业

| 地区 | 公司 | 上市 / 股价 |
|---|---|---|
| 国内 | [公司名](url) | <股价 badge 原样照抄> |
| 国内 | ... | ... |
| 国外 | [公司名](url) | <股价 badge 原样照抄> |
```

- **公司**列 = 原来的 markdown 链接 `[名字](url)`，原样照抄。
- **上市 / 股价**列 = 原来跟在链接后面的那段 badge HTML，**逐字符原样照抄，绝对不要改动**。它有三种形态，都直接搬：
  - `<span class="sq" data-stock="hk:02513"></span>`
  - `<span class="sq-none">未上市</span>`
  - `<a class="sq-na-link" href="..." target="_blank" rel="noopener">行情↗</a>`
- 顺序：先所有「国内」行，再所有「国外」行，各自保持原顺序。
- 删掉 `**国内**` / `**国外**` 这两个粗体小标题（信息已进「地区」列）。

## 2. `### 科研院所` → 单表（地区作列）

```
### 科研院所

| 地区 | 机构 | 研究重点 |
|---|---|---|
| 国内 | [机构名](url) | 原来 — 后面的描述文字 |
| 国外 | [机构名](url) | 描述文字 |
```

- **机构**列 = 原 markdown 链接原样照抄。
- **研究重点**列 = 原来 `— ` 破折号后面的描述，原样搬（去掉前导的「— 」）。
- 同样先国内后国外、保持原顺序、删掉粗体小标题。

## 3. `## 顶会顶刊` → 逐条表（名称 + 说明）

原来是 `**顶会**：A · B · C` 和 `**顶刊**：X · Y` 两行。改成：

```
## 顶会顶刊

| 类型 | 名称 | 说明 |
|---|---|---|
| 顶会 | NeurIPS | 神经信息处理系统大会 |
| 顶会 | ICML | 国际机器学习大会 |
| 顶刊 | TPAMI | IEEE 模式分析与机器智能汇刊 |
```

- 把 `·` 分隔的每一项拆成一行；顶会的行 `类型`=顶会，顶刊的行 `类型`=顶刊，保持原顺序。
- **名称**列 = 该会议/期刊的**缩写/简称**。
- **说明**列 = 全称或中文释义：
  - 若原文该项已自带括号注释（如 `ECTC（电子元件与技术大会）`），把 `ECTC` 放名称列、括号里的内容放说明列。
  - 若原文只有缩写没有注释（如 `NeurIPS`、`ISSCC`、`IEDM`、`DAC`），在说明列补它**公认的标准全称/中文译名**（这些都是知名学术会议/期刊，用准确的标准名称）。
  - **不确定某个会议/期刊的确切全称时，宁可留简短中文领域说明，也不要编造**。

## 完成后

- 你的汇报回传给主控，不展示给用户。
