# 任务简报 v2：科研方向「代表性机构」按子方向补全（修正漏检）

你是本仓库（MkDocs ECE 自学指南）的内容编辑 agent。上一轮已经把每个方向的 `## 代表性机构` 改成了「企业/科研院所 × 国内/国外」分级列表 + 独立的 `## 顶会顶刊`。**本轮是质量补全**，修正一个系统性问题：上一轮**没有先精读「这个方向在研究什么」就去搜企业，导致漏掉了该方向的某些子方向对应的企业**。

## 失败案例（务必理解这个教训）

`光电子与硅光集成` 的 `## 这个方向在研究什么` 用一整节讲 **光计算（optical computing / 光子神经网络 / MZI 矩阵乘法）**，明确提到 Lightmatter、清华太极。但上一轮的企业列表只有光模块/光互连/CPO 公司（中际旭创、新易盛、Intel、Coherent…），**完全漏掉了光计算这条子方向的代表企业**（如 Lightmatter、Lightelligence 曦智科技、Celestial AI、光本位 等）。

这就是「望文生义、按方向名搜索」的后果——方向名叫"硅光集成"，就只搜了硅光通信/光模块，漏了同样写在正文里的"光计算"。

## 本轮方法（强制按此顺序）

对你负责的每个方向：

1. **先完整读 `## 这个方向在研究什么` 整节**（连同里面的 SVG 文字、加粗术语、"前沿""值得一提"段落都要读）。
2. **列出这个方向正文里出现的所有子方向 / 研究分支**（包括应用分支、前沿分支、不同技术路线）。例如光电子至少有：硅光通信/光模块、共封装光学 CPO、光互连/光 I/O、**光计算/光子神经网络**、（可能还有）光量子/传感等。
3. **逐个子方向去搜企业**（WebSearch/WebFetch，中美为主，其他重要海外酌情）：每个子方向至少要有代表企业被收录（若该子方向确实没有对应企业，如纯学术前沿，则在院所里体现）。
4. 把**漏掉的子方向对应企业补进现有列表**；现有已验证的条目保留；明显不相关或链接失效的删掉。
5. 科研院所同理——按子方向检查是否有代表院所/实验室漏掉。

## 自检要求（必须在汇报里写出来）

汇报时，对每个方向给出「子方向 → 对应企业」的映射清单，证明每个子方向都被覆盖了。例如：
```
光电子与硅光集成：
- 硅光通信/光模块 → 中际旭创、新易盛、Coherent、Lumentum ✓（已有）
- 共封装光学 CPO → 天孚通信、博创科技、Marvell ✓
- 光互连/光 I/O → Ayar Labs、Intel ✓
- 光计算/光子神经网络 → Lightmatter、Lightelligence(曦智)、Celestial AI、光本位 ★新增
```

## 版式 / 股价规则（与上一轮一致，不要改变已成型的版式）

保持现有结构：`## 代表性机构` →（`### 企业` →`**国内**`/`**国外**`；`### 科研院所` →`**国内**`/`**国外**`）→ `## 顶会顶刊`。每条单独换行、带官网超链接；上市公司附实时股价链接，未上市标注。

- A股·上交所：`https://quote.eastmoney.com/sh<6位>.html`；深交所：`.../sz<6位>.html`；北交所：`.../bj<6位>.html`
- 港股：`https://quote.eastmoney.com/hk/<5位>.html`
- 美股：`https://finance.yahoo.com/quote/<TICKER>`；其他海外用 Yahoo 带后缀（韩 .KS / 日 .T / 台 .TW；欧洲优先 ADR）
- 未上市 / 母公司未上市：不给股价链接，官网链接后纯文本注明 `（未上市）` 或 `（XX 未上市）`；若是上市母公司的部门，给母公司股价并注明。

## 正确性约束（不可妥协，作者反复强调「不要幻觉乱编」）

1. 新增/保留的每个官网 URL 必须真实可访问，WebFetch/WebSearch 核实（DNS 能解析、非 404/停靠页/失效域名）。注意上一轮就出过两处失效域名（quantum.baidu.com 已停、成都华微 csmsc.com 域名失效），不要重蹈覆辙。
2. 股票代码必须真实，核实交易所与代码再拼 URL。
3. 公司/院所必须**确实做这个子方向**——以其官网/产品页/Scholar 为准，禁止凭名字猜。
4. 院所列机构层面（国家实验室/研究所/公司研究院/高校实验室中心），不要和页面后面 `## 相关课题组` 的教授名单重复。
5. 去重：同一主体不要重复出现。
