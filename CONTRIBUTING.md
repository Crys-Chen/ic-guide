# 贡献指南

感谢你愿意完善 IC Guide。这是一份面向集成电路学习者的自学地图，欢迎直接发 PR；如果暂时不熟悉 GitHub，也可以开 Issue 或在页面底部留言。

## 🌟 优先贡献什么

- 修正错别字、死链、过时信息和事实错误。
- 补全低置信度科研方向页、空白课程页、论文推荐和科普视频。
- 为已有页面补充自己实际学过、做过、查验过的内容。

## 🛠 发 PR 的基本流程

1. 登录 GitHub，打开要修改的文件。
2. 点击右上角铅笔图标编辑文件，GitHub 会自动 fork 仓库。
3. 修改后点击 `Commit changes...`，选择 `Create a new branch`。
4. 点击 `Propose changes`，再创建 Pull Request。

如果会在本地开发，也可以 fork 仓库后用常规 Git 流程提交 PR。提交前请尽量保持改动集中：一类问题一个 PR，不要把无关格式化、批量重排和内容改写混在一起。

## ✏️ 页面编辑原则

- 沿用同类页面的现有结构、语气和 Markdown 写法。
- 事实性内容优先引用官方主页、课程主页、论文、Google Scholar、会议/期刊页面或公开可访问资料。
- 不确定的信息宁可少写，不要为了凑完整度而扩大表述。
- 关键词尽量写成 2-3 个短语，避免宣传语和无法从来源中验证的概括。
- 不接受产品推广或付费推广。

## 💡 补充教授、课题组或企业

补充到科研方向页时，请先确认三件事：

1. **归属方向贴切**：主页、论文、项目或公开资料能直接支撑该对象属于该科研方向。
2. **链接合适**：优先使用本人个人主页、课题组主页、学校官方个人页或企业官网。不要链接宽泛的学院门户页、新闻聚合页、搜索结果页；演讲/报告列表只能作为佐证，不应替代主页链接。
3. **关键词可验证**：子方向关键词必须能从主页、论文、Google Scholar 或官网材料中看出，不要自行拔高。

示例格式：

```markdown
-   **[姓名](https://example.edu/profile)** <span class="badge-fudan">复旦</span>

    关键词一 | 关键词二 | 关键词三
```

如果证据只显示“相关”而不是“主要从事”，请把位置放在更保守的区域，措辞也相应收窄；证据不足时先开 Issue 讨论。

## 📄 新建课程页

课程页按站内已有课程页结构编写。文件命名规则：

- 有课号的课：`校名缩写_课号.md`，如 `MIT_6.042J.md`、`FDU_MICR130008.md`。
- 无课号的大学课：`校名缩写_教师拼音.md`，否则用 `校名缩写_主题.md`。
- 个人或平台创作者：`作者_主题.md`，如 `karpathy_zero2hero.md`。

分隔符只用下划线，不用空格和全角字符。

新建课程页时，通常还要同步更新：

- 父级 `index.md` 的子目录或课程列表。
- `mkdocs.yml` 里的 `nav`，否则页面不会出现在导航中。

如果新建的是完整子分类，还需要更新上一级 `index.md` 中的 mermaid 知识谱系图和板块介绍。拿不准时可以在 PR 说明里写“上层 index 和 nav 请维护者处理”。

## 🌌 新建科研方向页

科研方向页比课程页复杂。除内容页外，通常还要同步更新：

- `docs/科研方向/index.md`：方向列表和全景图。
- `docs/javascripts/orbit-galaxy.js`：`DIRECTIONS` 与 `CARDS` 配置。
- `mkdocs.yml`：`科研方向` 导航。

`orbit-galaxy.js` 的 `ring` / `angle` 需要手动调位置，避免节点重叠。建议先开 Issue 确认方向和定位，再写内容页。

## 📬 推荐内容但不直接改文件

也可以通过 Issue 提交：

- 补充教授或企业：选择归属方向，提供姓名/名称、机构、主页 URL 和 2-3 个可验证关键词。
- 新建学习地图知识版块：说明上级板块、版块边界、候选课程资源，以及是否需要维护者协助更新 `index.md` 和 `mkdocs.yml`。
- 推荐论文：提供归属方向、标题、DOI 或公开链接、推荐类型和推荐理由。
- 推荐科普视频：提供平台、链接、标题和适合人群，内容必须公开免费可看。
- 投稿笔记：说明课程或方向、笔记类型和版权情况，可上传 PDF / zip 或提供外链。

## 🔍 本地检查

如果本地安装了依赖，提交前可以运行：

```bash
mkdocs build --strict
```

没有本地环境也可以提交 PR，但请至少检查链接是否能打开、Markdown 列表缩进是否与周围一致。
