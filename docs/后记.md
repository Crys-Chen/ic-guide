# 后记

1. 本科教育不行
2. 浩如烟海，科研方向晕字。学术圈
3. 信息差
3. 开源精神

最初北大的CS自学指南，是我在复旦系际辩论赛打辩论时了解到的。那道辩题我记得是，“当今中国，加强通识教育/专业教育更有利于大学生未来发展”。我方是专业学习。有一条论证路径是“缺啥补啥”，当今中国内地的本科教育，无法给人才应有的教育，连北大都需要“CS自学指南”来自救。

> 本科教育至少得展现它应有的诚意，一种分享知识的诚意，一种以人为本的诚意，一种注重学生体验的诚意。它至少不应该是一种恶意，一种拼比知识的恶意，一种胜者为王的恶意，一种让人学无所得的恶意。但这一切能改变吗？我不知道。————《CS自学指南》

> 国内绝大部分大学的本科教学，不是濒临崩溃，而是早已崩溃。————《上海交通大学学生生存手册》

从最初有这个想法开始，这个网站断断续续做了两年，断断续续。最初只是想搭一个个人网站，分享一些高年级的认知和学习笔记，仅局限于复旦校内。慢慢地我觉得，这样没意思。在这个专业里受苦受难的不止复旦的学子。如果一个从社交媒体慕名而来的同学，点开网站，发现这些资源都是复旦专属，那该有多失望！

“我想成为的人”和“系统引导我成为的人”之间存在着很大的张力。有些管教育的，嘴上说要培养什么样的人，实际上他们制订的制度，设计的培养方案，以及相应的激励机制，南辕北辙。单纯从培养方案来看，从系统上来看，就是要我们面面俱到。可“生也有涯，知也无涯”，如果我照着系统的要求，每一门课都付出十足的精力去学，去拿高分，那我的生活怎么办？

“弱水三千，只取一瓢”

搭建网页对我来说很困难，学习前端的知识，UI的设计，要用衬线字体还是非衬线字体，按钮要圆角还是？非常繁琐，做一个轮播图，就过去了一天。直到claude code的出现。

我记得去年暑假，我还在坚持手撸代码。那时的AI会写代码，但写完需要人类去debug，整体效率没有提升。今年已经可以自动搭建一个网页。但还是比较难受的。无论是画图还是做网页，AI都只能快速搭建一个80%的东西，剩下20%，如果完全不懂web开发，不懂代码，就只能用prompt去调，这同样是旷日持久的工程。

程序员这个群体太共产了，开源精神很好，研发出AI，第一件事情竟然是革自己的命。相比之下，做电路的一直在闭门造车。CS大量的自学资源，是EE难以企及的。在前AI时代，某个科研方向到底在做什么，往往只能靠搜索引擎极其有限的信息、学长姐的口口相传，外加实践出真知获得。

90%的授课老师，都没有网课老师教得好；99%的助教，都没有AI耐心。我们当下确实在经历着教学方式的变革。随着时间的流逝，我这个网站的意义也在慢慢下降。但没关系，大家也可以直接用AI扫一下本站，再制订自己的学习路径，如此，可以比无脑提问节省很多token。

“本科教育”是什么？“本科”与“教育”两个词，可能已经不适合再放在一起看。据我所知，只有学工那群做宣传口的还在琢磨这种抽象的东西。一个理想的大学就应该是一个自助餐厅，尽可能多的给钱给资源，我们身处其中，各取所需。不要想着教学生成为什么样的人。

“国内绝大部分大学的本科教学，不是濒临崩溃，而是早已崩溃”————这句话诞生于2008年。敢问今夕是何年？现在的本科教育相较于当年如何，诸君冷暖自知。世道如此，倒也不必苛责系统里处于某个位置的人。或许“本科”与“教育”这两个词已经不再适合放在一起思量。大学之于学生，自当是“弱水三千，只取一瓢”。

跟他不同的是，我无意改变什么。设计这个网站让我很开心；把在脑海里积攒多年的信息倒出来让我很开心；把每一个科研方向用讲故事的方式说清楚很开心；写下这段文字让我很开心；如果屏幕前的有缘人发现觉得网站里的东西有用，那我也很开心。

我走到哪，就把路铺到哪。
