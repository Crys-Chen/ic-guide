## 文风标准（每篇对照自查）

**硬规矩（已在 memory）**
- **翻译腔零容忍**：不要被动句套从句、不要"具有…等优点""它的存在提示""至少提醒一件事"这类直译腔。正统中文、短句、句号推进。
- **破折号/冒号/括号克制**：段落里做解释的 `——` 和 `：` 尽量 0 个、最多 1 个；想写"X：Y""X——Y"就拆成两句用句号断开。括号注释每段 ≤1，只留必要术语缩写。详见 memory `writing-punctuation-rule`。
- **不点名教授**（正文/适合谁），公司须核心业务对口（memory `career-destination-rule`）。

**软毛病（最常见的"文风不行"，逐条排查）**
- **枚举病**：A 做 X、B 做 Y、C 做 Z 的清单感，没有叙事线索。
- **新闻稿大词**：质变、推到极致、卡脖子、力大砖飞、已经走通、不可替代的位置。
- **meta 自评句**：".. 是最重要的演进""这件事的意义在于…"——删包装，直接给事实。
- **学院腔过渡**："学术界做研究的可行路径是…""应用拓展上 XX 是另一条线索"。
- **装饰句**：对大一大二没用、只增阅读负担的话（流程细节、自我表扬、学术圈八卦）全删。
- **段落过渡靠内容驱动**，不要"下面五段…""上面讲的是…"这种元层自指。
- 详细踩坑清单见 `todo.md` 的「AI 算法与系统迭代教训」section。

**自检 checklist（改完必走一遍）**
1. 每句对大一大二有用吗？装饰/概括废话/meta/流程细节 删。
2. 每段有论证骨架（立论→原理→佐证→收束）还是话题罗列？
3. 一段里破折号/冒号/括号出现 3+ 次？
4. 段间有重复表达吗？
5. 翻译腔/大词/学院腔/中英混杂？
6. 字句逻辑有断裂或矛盾吗？

## 工作流

1. 用户说「改 X」→ 我先**通读 X 正文，逐段诊断文风病灶**（指出哪句哪段、什么病）。
2. 提**修改方案**（不动叙事大纲，除非用户要求重建）。
3. 用户确认 → 改 → 自己 critically 读一遍 → `mkdocs build` 验证。
4. 更新本表状态与备注。**不主动开下一篇。**
