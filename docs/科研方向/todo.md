## 写作规则

### 内容要求
- 用具体的 **Hook** 开头（一个让读者有画面感的场景或产品）
- 有**比喻或类比**帮助读者快速理解核心概念
- 覆盖该方向的主要研究内容和流派
- 适当给关键术语**加粗**
- 整篇文章**不能有小标题**，用 prose 写作
- 整篇文章要有**明确的叙事线索**，像讲故事一样带读者过完整个方向，不能只是无聊的枚举
- 在适当的地方要有**图示**，让读者一目了然——可以用 SVG 画，也可以直接用著名论文的配图

### 写作风格
- 严格避免英译中的翻译腔，用正统中文叙述
- 有个性，不能死板
- 不能变成枯燥的罗列
- **禁止过分使用破折号、冒号、括号等不符合中文表达习惯的注释方式**——中文 prose 应当多用句号断句,而不是靠破折号/冒号/括号串句

## AI 算法与系统迭代教训(每次写作必读)

> 这一节是 AI 算法与系统改写过程中,经过用户多轮严厉反馈得到的具体行为准则。**每次开始一个新方向前,扫一遍这一节;写完后用底部的自检 checklist 自查后再交付**。

### 大纲组织
- **主线必须是真问题,不是分类目录**:避免"按 X 分类展开 + 主线之外"的清单感。要找一个核心问题(如"算力撞墙之前怎么变强"、"数据搬运的代价"、"换一种物理顶上去")驱动整段
- **每段必须有论证骨架**:立论 → 解释原理 → 数字佐证 → 收束。不要话题罗列
- **段落过渡靠内容驱动,不是元层自指**:
  - ❌ "下面五段..."、"上面三段讲的是..."、"应用拓展上,XX 是另一条线索"、"代表性工作:..."
  - ✅ 直接以内容判断/事实进入(范本:存算一体的"研究者走出了两条截然不同的路")
- **逻辑顺序**:thesis → why → 具体证据 → solution。不是 thesis → solution → why → problem(断裂)
- **重复表达**:thesis 已经分了 X 和 Y,后面不要再说"业界在 X 和 Y 两条线上努力"

### 文风具体禁忌
- **拒绝翻译腔**:"至少提醒一件事——"(直译 reminds us that)、"它的存在本身就提示"、"和平共处"、"舒适区"、"具有...等优点"
- **拒绝 meta 评价句**:"L2 的兑现是 2024 年最大的范式转变"、"这件事的意义不在某个具体模型,而是..."、"是当前最重要的演进" → 删 meta 包装,直接给事实让读者自己判断
- **拒绝新闻稿大词**:"推到了人类工程史上最复杂"、"质变"、"收编"、"已经走通"、"独特的、不可替代的位置"
- **拒绝学院腔过渡**:"学术界做研究的实际可行路径,是..."、"应用拓展上 XX 是另一条线索"
- **拒绝网络流行语**:力大砖飞、卡脖子、稳稳接住、客串(除非用户主动用了具体观察句)
- **拒绝 inventory 列表风格**:名词箭头链(RLHF→DPO→GRPO→PRM)、连续产品列表带括号注释

### 标点克制(用户原话:"不要老是括号注释,非常死板")
- 一段里 **3+ 破折号 → 用句号断**
- **括号注释 2+ → 改成自然句子**(用"比如 X"引出,或断句)
- 长串术语堆砌不解释(如"数据/张量/流水线并行 + ZeRO/FSDP")→ 压缩成一个核心概念

### 中英混杂边界
- **保留英文**:专有产品名(Claude Code、Flash Attention、Llama)、特定缩写(Transformer、attention、MoE、HBM、KV Cache、SAE)
- **必须中文化**:可中文化的(scaling axis、test-time compute、paradigm shift、self-evolve、reasoning trace、Foundation models for science、AI safety、frontier)
- **翻译保留**:Constitutional AI → 宪法 AI、Interpretability → 可解释性

### 占位符填空 vs 重写(关键边界)
- 用户标 `??` 或留占位符("Anthropic 的'宪法'"、"再举一些例子") = **简短填入一句**,不要展开成段
- 用户改写整段 = 大纲已锁定,**不要再大改结构**,只改具体字句
- **列清单让用户确认 → 然后只改用户同意的项目**,不要顺手扩

### fact-checking 优先(最严重的踩坑)
- **任何 present-day fact 必须 web_search 先**,不要靠 priors 否定具体事件/产品/术语
- 用户提到了你没听过的术语/事件/产品 = 立刻搜,不是先问 / 不是先猜
- AI 算法与系统过程中我曾否定 OpenClaw、新版 Codex、Claude/Minab 事件 — **全是真的**。这是最严重的错误

### 读者校验尺
对每一句话问:**"这句话对大一大二有没有用,会不会反而给他们增加阅读负担?"**
- 流程细节("通过 Bloomberg")、自我评价("这是过去十多年里第一次...")、学术圈八卦(NeurIPS/ISSCC 双投录用)、产品括号注释 — **全是装饰,删**
- 但具体物理图像(KV Cache 是啥、悬挂键、DIBL)和锚点数字(220 万晶体 vs 4.8 万)要保留

### 自检 checklist(写完必读,用户原话"不要直接把一坨狗屎端给我")
1. 每句话对大一大二有用吗?装饰句 / 概括废话 / meta 评价 / 流程细节 全删
2. 每段有论证骨架吗?还是话题罗列?
3. 标点密集吗?破折号 / 冒号 / 括号在一段里出现 3+ 次?
4. 段落之间有重复表达吗?
5. 中英混杂、新闻稿大词、翻译腔、学院腔有吗?
6. 字句逻辑有断裂或矛盾吗?(典型如"Transformer 对 GPU 友好"紧接"Llama 训了 54 天"读起来像打架)

### 工作流偏好
1. **先讨论大纲 → 再动笔**(不要直接交一坨)
2. **改文风 ≠ 改大纲**(除非用户主动要求重建)
3. **列清单让用户确认 → 然后只改同意项**(用户最后这轮要求的方式)
4. **写完自己 critically 读一遍再交**

## 诊断标准

好的开头应该让读者产生"原来是这么回事"的感觉，而不是"又在背课文"的感觉。核心病症有三类：
- **枚举病**：A 做 X，B 做 Y，C 做 Z……像产品说明书，没有叙事逻辑
- **翻译腔**：句式被动、从句嵌套、用"具有……等优点"这类模板
- **缺少锚点**：通篇技术术语但没有一个让人"啊这样啊"的具体场景或物理图像

按需改写的优先级分三档：**🔴 需要重写** / **🟡 需要局部手术** / **🟢 基本OK，微调即可**

- **技术路线段是全项目最严重的枚举**：超导/离子阱/光量子/硅基量子点四种路线整齐并列，像产品对比表
- 产业格局段（IBM→Google→中科大→本源量子→国盾量子）也是枚举

**改写方向**：选一种路线（超导）讲深——约瑟夫森结为什么能当量子比特？为什么要冷到 20 mK？其他三种用"替代路线"简要对比。产业格局砍掉或移到表格里。

### 16. 类脑芯片
**问题诊断**：
- 开头（矩阵乘法 vs 脉冲）的对比框架是对的
- **TrueNorth→天机的描述像新闻稿**，列参数不是理解
- 忆阻器段和存算一体章节高度重复

**改写方向**：深入解释"事件驱动"为什么省能——大多数时候大多数神经元根本不工作。TrueNorth 和天机讲解决了什么/还没解决什么，不要列参数。忆阻器段只保留类脑场景的独特价值（模拟突触可塑性），不重复 CIM 基本原理。

### 17. 硬件安全与可信计算 ✅ 已完成
**问题诊断**：
- Spectre 开头非常好，有故事、有冲击力
- **紧接着三段是标准的子领域罗列**：侧信道一段、硬件木马一段、PUF 一段，三段并列就是一份大纲
- "ARM TrustZone 和 RISC-V PMP 等可信执行环境（TEE）"——一口气列三个缩写

**改写方向**：Spectre 开头保留。三个子领域用"攻击面从哪来"串成因果链——性能优化留下侧信道，供应链信任缺失留下硬件木马，防御的根基是物理不可伪造性（PUF）。三段变因果链而不是清单。
