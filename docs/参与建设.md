## 怎么贡献

### 随手改错

发现错别字、死链、教授信息过时、事实错误，直接在**页面底部评论区**留言（需要 GitHub 登录），或者[开一个 Issue](https://github.com/Crys-Chen/Fudan-ME/issues/new)，一句话描述就够。

### 补充教授或企业

[填写表单 →](https://github.com/Crys-Chen/Fudan-ME/issues/new?template=add_entry.yml)

选好归属方向，填姓名/名称、主页 URL、子方向描述，提交后我来处理。唯一硬约束：URL 必须能打开，且确实是本人个人主页或企业官网，不接受学院门户页。

### 完善现有页面某一节

某方向的"核心研究问题"只有两条、某课程页只有壳没有内容——[开 Issue 说一句](https://github.com/Crys-Chen/Fudan-ME/issues/new)你想改哪节、改成什么，或者直接发 PR。按页面已有结构写就行，不需要提前对齐。

### 新建课程页

按[课程页模板](https://github.com/Crys-Chen/Fudan-ME/blob/main/template.md)写，发 PR。

文件命名规则：

- 有课号的课：`校名缩写_课号.md`，如 `MIT_6.042J.md`、`FDU_MICR130008.md`
- 无课号的大学课：教师是卖点时用 `校名缩写_教师拼音.md`，否则用 `校名缩写_主题.md`
- 个人 / 平台创作者：`作者_主题.md`，如 `karpathy_zero2hero.md`

分隔符只用下划线，不用空格和全角字符。

### 新建科研方向页

建议先[开 Issue 说明你想做哪个方向](https://github.com/Crys-Chen/Fudan-ME/issues/new)，确认范围再动手，免得写完发现不对路。方向页有七节结构，第一节（这个方向在研究什么）最难写，模板里有说明。

### 分享笔记

[填写表单 →](https://github.com/Crys-Chen/Fudan-ME/issues/new?template=submit_notes.yml)，文件直接拖进 Issue 正文（PDF / zip，单个 ≤ 25 MB）。笔记放在自己网盘或仓库的，PR 里加一行链接也行。投稿经审核后会挂到笔记索引页。

## 三条底线

**不能编。** 教授和企业的主页 URL 必须能打开，且确实是本人/本公司的页面。查不到就不填，不要靠猜。

**用了 LLM 就自己重写一遍。** AI 给的"丝滑过渡句"和"完美总结段"是这个站最反对的语气。

**批制度，不苛责个人。** 批评培养方案、课程设置、行业现状都 OK；指名道姓批评某个具体的人不 OK。

## 一点说明

不接受产品推广或付费推广。课题组招生、企业招聘等免费社区信息视站点发展情况另议。

愿意署名就署名，想匿名也完全 OK。
