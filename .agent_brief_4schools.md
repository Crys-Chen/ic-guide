# 作业规范：为科研方向页补充「上交 / 浙大 / 中科大 / 南大」教授

你是一个被派发来处理 **单一科研方向页** 的 agent。你的任务：对**上海交通大学、浙江大学、中国科学技术大学、南京大学**四所学校的工科院系做**地毯式搜索**，找出在你负责的这个方向上有**真实、可验证**工作的教授，按规范插入该方向页的「境内」段。

## 0. 绝对红线（不可妥协）

1. **禁止幻觉/编造**。每位教授必须真实存在、且其研究确实属于这个方向。子方向描述必须基于其**实际研究**（个人主页 / 课题组页 / Google Scholar），不得仅凭姓名+方向猜测。
2. **每个链接必须 WebFetch 验证可访问**（HTTP 200，且页面内容确实是该教授本人）。链接打不开或内容对不上 → 换一个权威页面（院系师资主页 / 课题组页 / Google Scholar），仍找不到可靠链接就**不收录**该教授。
3. **不要重复添加已有的教授**。先把本文件「境内」「境外」两段里已存在的教授名单读出来，按姓名 + 主页 URL 去重。
4. **只动你负责的这一个文件**，且**只在「境内」段插入**（这四校都是境内）。不要碰「境外」段，不要碰正文其它部分。

## 1. 先吃透方向内涵，再开始搜（最重要，别跳过）

**这是质量的根本。** 在动手搜索前，你必须先把方向页里这几节**完整读一遍并真正理解**：
- `这个方向在研究什么`（核心，往往很长，讲清了这个方向的真实边界、子问题、技术脉络）
- `核心研究问题`
- `代表性机构`（含国内外**企业**与机构——这是判断"什么算这个方向"的最好标尺）
- `知识路径` / `适合什么样的人`

读完后，先用自己的话写下这个方向的**真实研究范围**（覆盖哪些子方向、不覆盖哪些临近但不属于的东西），再去搜。

**两个必须避免的失败模式：**
1. **望文生义（按关键词字面匹配）**：任务里给你的关键词只是**线索/提示**，不是过滤器。不要看到某人主页有某个词就收、没有就弃。要看其**实际研究是否落在你上面总结的方向范围内**。很多对口的人用的术语和关键词不同（换了说法），死扣关键词会**误收**字面命中但实质无关的人。
2. **搜得太浅会漏**：因为只盯关键词，会漏掉用不同表述、但确实在做这个方向的人。要顺着"这个方向在研究什么"里提到的**具体子问题/技术/代表工作**去反查国内有谁在做，再定位到四校的人。

## 2. 搜索范围（地毯式 = 尽量穷尽相关院系）

按方向相关性，逐个排查以下院系的**教师名录页**，逐人对照你上面总结的方向真实范围（不是字面关键词）。不同方向重点院系不同，自行判断哪些院系相关：

**上海交通大学**：电子信息与电气工程学院（微纳电子学系 / 电子工程系 / 计算机科学与工程系 / 自动化系 / 信息与通信工程系 / 感知科学与工程学院）、集成电路学院、人工智能学院、物理与天文学院、材料科学与工程学院、机械与动力工程学院、生物医学工程学院、Wu Wenjun 人工智能荣誉博士班相关 PI。

**浙江大学**：信息与电子工程学院（ISEE）、微纳电子学院、计算机科学与技术学院、控制科学与工程学院、集成电路学院、光电科学与工程学院（光电强）、物理学院、材料科学与工程学院、生物医学工程与仪器科学学院（脑机接口强）。

**中国科学技术大学**：信息科学技术学院（电子工程与信息科学系 EEIS / 自动化系）、微电子学院、计算机科学与技术学院、物理学院 / 中科院量子信息重点实验室（量子超强：潘建伟、郭光灿、杜江峰等）、合肥微尺度物质科学国家研究中心、工程科学学院、网络空间安全学院。

**南京大学**：电子科学与工程学院（微电子学与固体电子学 / 电子科学与技术 / 通信工程）、计算机科学与技术系 / 软件学院、人工智能学院（周志华团队）、智能科学与技术学院、物理学院（固体微结构物理国家重点实验室，半导体物理强）、现代工程与应用科学学院（MEAS，材料/光电/能源）、集成电路学院（苏州）。

## 2. 数量与质量

- **质量优先于数量**。只收录在这个方向上有公认/可验证工作的人，宁缺毋滥，不要为凑数塞弱相关的人。
- 某校在某方向若确实没有合适的人，**就不收录该校**，这是正常的。
- 典型每校 0~6 人，视该校在该方向的实力而定。

## 3. 卡片格式（严格照抄）

每位教授一张卡片，放在 `<div class="grid cards prof-collapse" markdown>` 列表里：

```
-   **[姓名](已验证的主页URL)** <span class="badge-other">交大</span>

    子方向1 · 子方向2 · 子方向3
```

- **badge 文字**：上海交通大学→`交大`，浙江大学→`浙大`，中国科学技术大学→`中科大`，南京大学→`南大`。四校**统一用 `class="badge-other"`**。
- **女教授**：在 badge 后追加 `<span class="prof-w"></span>`（务必准确判断性别，男教授不加）。例：
  `**[张三](url)** <span class="badge-other">浙大</span> <span class="prof-w"></span>`
- **子方向**：2~4 个，用 ` · ` 分隔，中文为主、可夹英文术语，须反映其真实研究。
- 姓名链接的中文名后若常用英文名，可写 `中文名（拼音/英文名）`，但非必须。

## 4. 插入位置与排序

- 插入「### 境内」段的 `<div class="grid cards prof-collapse" markdown>` ... `</div>` 列表**内部**。
- 放在「其他国内」簇：即在 清华/北大/复旦 各卡之后、`</div>`（境内段结束）之前。若该段已有交大/浙大/中科大/中科院/南大等条目，把新卡**与它们相邻合并**。
- 同簇内相对顺序：**交大 → 浙大 → 中科大 → 南大**（已有的中科院条目保持其原位即可，不强行调整）。
- 卡片之间用一个空行分隔，与现有卡片风格一致（注意缩进：`-` 顶格，描述行前有 4 个空格）。

## 5. 完成后

- 用简短文字汇报：你为这个方向新增了哪些教授（姓名+学校+一句方向），每校各几人；哪些校无合适人选；列出你验证过但**舍弃**的（链接失效/方向不符/疑似重复）。
- 你的汇报文本会回传给主控，不展示给用户，所以直接给结构化结论即可。
